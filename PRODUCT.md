# Product

## Register

product

## Users

Minecraft Java, Bedrock, and MCPE players arriving from urgent searches for a current Verity Mod download, installation help, source verification, or a fix. They need a trustworthy answer before opening files or changing a game profile.

## Product Purpose

Help a player identify the correct Verity project and package, distinguish Java from Bedrock routes, inspect source and filename signals, and continue to a traceable publisher page. Success means users can make the next installation decision quickly without trusting a mirror, exposing credentials, or mistaking an identity check for a malware guarantee.

## Brand Personality

Evidence-driven, direct, cautious. The interface should feel like a clear technical workbench with enough Verity character to be recognizable, while keeping status, labels, and controls unambiguous.

## Anti-references

- Fake-download pages with several competing primary buttons, hidden destinations, countdowns, or installer wrappers.
- Horror pages so dark or atmospheric that users cannot inspect the actual package and source information.
- Decorative overlaps that crop headings, labels, status rows, or important project metadata.
- SEO pages that repeat the keyword without helping users choose an edition, package, or next action.

## Design Principles

- Keep every current status and project identity fully readable.
- Make the edition and package decision before presenting installation details.
- Separate verified identity signals, cautions, and danger states with explicit language.
- Prefer traceable publisher records over direct file hosting.
- Preserve a stable layout at common desktop and mobile viewport sizes.

## Accessibility & Inclusion

Target WCAG 2.1 AA contrast and keyboard operation. Do not communicate state by color alone. Respect reduced-motion preferences, keep focus indicators visible, and ensure headings, controls, metadata, and long fingerprints remain readable without horizontal scrolling.
